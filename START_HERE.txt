BULSHO STORE — SUPABASE EDITION

1. In Supabase Dashboard, open SQL Editor and run:
   store-backend/supabase/schema.sql

2. Backend setup (PowerShell):
   cd store-backend
   Copy-Item .env.example .env
   npm install

3. Edit store-backend/.env and set:
   SUPABASE_DB_URL=<copy from Supabase Connect>
   JWT_SECRET=<random 32+ character value>
   REFRESH_TOKEN_SECRET=<different random 32+ character value>

4. Verify and run backend:
   npm run db:verify
   npm run seed:admin
   npm run dev

5. Frontend setup in a second terminal:
   cd store-frontend
   Copy-Item .env.example .env
   npm install
   npm run dev

6. Optional old MongoDB data import:
   Read SUPABASE_SETUP.md, section “Optional MongoDB data migration”.

IMPORTANT:
- Never put SUPABASE_DB_URL in the frontend.
- Never commit real .env files.
- The UI and REST endpoints are preserved; only persistence changed.
- A real Supabase project was not available during offline validation, so run db:verify before use.
